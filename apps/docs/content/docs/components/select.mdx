## Displaying labels in the trigger

`SelectValue` renders the selected option's `value` by default (e.g. `"us"`).
To show the label (e.g. `"United States"`) instead, pass an `items` map to
`Select` so it can resolve value ‚Üí label:

```tsx
const countries = { us: "United States", ca: "Canada", uk: "United Kingdom" };

<Select items={countries} defaultValue="us">
  <SelectTrigger>
    <SelectValue />
  </SelectTrigger>
  <SelectContent>
    {Object.entries(countries).map(([value, label]) => (
      <SelectItem key={value} value={value}>{label}</SelectItem>
    ))}
  </SelectContent>
</Select>
```

For dynamic or computed display, pass a render function as `SelectValue`'s
child. This is the pattern used by `<ComponentPreview name="select-multiple" />`:

```tsx
<SelectValue>{(value) => labels[value] ?? "Select‚Ä¶"}</SelectValue>
```

## API Reference

### Select

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `value` | `string` | - | Controlled selected value |
| `defaultValue` | `string` | - | Default selected value (uncontrolled) |
| `onValueChange` | `(value: string) => void` | - | Callback when value changes |
| `disabled` | `boolean` | `false` | Disables the select |
| `multiple` | `boolean` | `false` | Allow multiple selections |
| `items` | `Record<string, ReactNode>` \| `Array<{ value, label }>` | - | Value ‚Üí label mapping for `SelectValue` to render in the trigger |

### SelectTrigger

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `size` | `"default" \| "sm"` | `"default"` | Size variant |
| `className` | `string` | - | Additional CSS classes |

### SelectItem

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `value` | `string` | - | Item value (required) |
| `disabled` | `boolean` | `false` | Disables the item |
