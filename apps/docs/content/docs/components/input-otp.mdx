## API Reference

### InputOTP

Powered by Base UI's [`OTPField`](https://base-ui.com/react/components/otp-field) preview primitive.

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `length` | `number` | – | Required. Number of OTP input slots. |
| `value` | `string` | – | Controlled OTP value. |
| `defaultValue` | `string` | – | Uncontrolled initial value. |
| `onValueChange` | `(value, eventDetails) => void` | – | Fires whenever the value changes. |
| `onValueComplete` | `(value, eventDetails) => void` | – | Fires once all slots are filled. |
| `onValueInvalid` | `(value, eventDetails) => void` | – | Fires when entered text is rejected by sanitization. |
| `validationType` | `"numeric" \| "alpha" \| "alphanumeric" \| "none"` | `"numeric"` | Built-in validation. Use `"none"` together with `sanitizeValue` for custom rules. |
| `sanitizeValue` | `(value: string) => string` | – | Custom sanitizer (only used when `validationType="none"`). |
| `mask` | `boolean` | `false` | Mask entered characters. |
| `inputMode` | `"none" \| "text" \| "tel" \| "url" \| "email" \| "numeric" \| "decimal" \| "search"` | – | Virtual keyboard hint. |
| `autoComplete` | `string` | `"one-time-code"` | Applied to the first slot and the hidden validation input. |
| `autoSubmit` | `boolean` | `false` | Submit the owning form once the value is complete. |
| `name` | `string` | – | Field name for form submission. |
| `form` | `string` | – | Associates the hidden input with a `<form id>`. |
| `disabled` | `boolean` | `false` | Disable the input. |
| `readOnly` | `boolean` | `false` | Prevent user edits. |
| `required` | `boolean` | `false` | Mark as required. |
| `isLoading` | `boolean` | `false` | Show a spinner and disable the input while verifying. |
| `id` | `string` | – | The id of the first slot. Subsequent slots derive ids from it (`{id}-2`, `{id}-3`, …). |

### InputOTPSlot

Renders a single character `<input>`. Standard input props (including `placeholder`, `aria-label`, `type`) are supported. Each slot should expose its position via `aria-label="Character N of M"` so assistive technology can announce the focused slot.

### InputOTPSeparator

A decorative separator (rendered as `OTPField.Separator` with `role="separator"`). Defaults to a minus icon; pass `children` to customize.
